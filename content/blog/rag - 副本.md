🔧 技术栈  
Python / LangChain / Chroma / Streamlit / DashScope / RAG

📌 你做了什么（重点！）
- 设计并实现了一个完整的 RAG 项目骨架，拆分为文档加载、文本切分、向量化、向量库管理、检索、Prompt 组装、问答链路和前端展示等模块
- 实现了对 PDF、DOCX、CSV 三类文档的统一加载能力，并将 CSV 行数据转换为更适合语义检索的自然语言描述
- 基于 `RecursiveCharacterTextSplitter` 实现文本切分逻辑，并为每个 chunk 保留原始 metadata，同时补充 `chunk_id`、`chunk_index` 等调试字段
- 使用 Chroma 搭建本地持久化向量数据库，支持知识库索引重建、加载和本地持久化管理
- 封装 embedding 模型初始化逻辑，支持通过环境变量配置 DashScope 向量模型，降低业务逻辑与模型实现的耦合
- 实现基于 Chroma 的检索器封装，支持按 `top_k` 返回相关文档片段，并保留完整 metadata 供后续来源展示使用
- 设计中文 Prompt 模板，约束模型必须基于检索上下文回答，并在上下文不足时返回“根据当前知识库内容，暂时无法确定”，减少幻觉
- 实现问答主链路 `answer_question(query)`，串联 embedding、向量库加载、检索、Prompt 组装、LLM 调用和结构化结果返回
- 对回答结果中的来源进行结构化抽取和去重，支持展示文件名、文件类型、页码、province、year 等关键信息
- 使用 Streamlit 开发最小可用前端页面，支持提问、回答展示、来源展示以及侧边栏一键重建知识库索引
- 统一整理索引构建流程，抽取公共 pipeline，避免脚本调用和页面调用之间的重复逻辑
- 增强异常处理和启动提示，例如知识库未构建提示、数据库目录校验、环境变量缺失提示、模型调用失败提示等

📊 成果（加分！）
- 完成了一个可本地运行的企业知识库 RAG 最小可用版本，具备“文档入库 -> 检索 -> 问答 -> 来源展示”的完整闭环
- 支持 3 类常见企业文档格式接入：PDF / DOCX / CSV
- 支持本地持久化知识库构建与重复重建，便于开发调试和演示
- 前端页面可直接完成提问和索引重建，提升了项目演示效率和可用性
- 为后续扩展重排检索、权限控制、增量更新、多知识库管理等能力预留了清晰的模块结构
