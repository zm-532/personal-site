# Agent 设计的关键，不只是多轮对话

很多 Agent 展示更强调“会不会调用工具”，但在真实场景里，我更关注流程是否稳定、角色是否一致、输出是否可控。

在反诈模拟训练平台里，我把 Agent 看作一个可编排的工作流系统，而不是单一提示词包装器。

# 为什么我会引入状态流转

如果所有逻辑都堆在一个 Prompt 里，系统会很快遇到这些问题：

- 角色状态不稳定
- 对话目标容易漂移
- 不同阶段的行为边界难以控制

所以我更倾向于把对话拆成多个阶段节点，再通过条件路由推进流程。

# 一个更可维护的实现思路

## 对话控制层

负责定义当前阶段该做什么，决定系统是继续追问、切换策略，还是进入总结环节。

## 角色约束层

负责维持角色一致性，避免模型在长对话中逐渐偏离设定。

## 场景生成层

负责根据训练目标动态生成输入材料，让系统更接近真实互动。

# 我的判断标准

一个好的 Agent 系统，不只是“看起来智能”，而是：

- 逻辑能被拆解
- 状态能被追踪
- 输出能被约束
- 后续能被迭代

这也是我在做大模型应用时最看重的工程思路。
