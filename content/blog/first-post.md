# 为什么我要做个人作品集网站

对我来说，个人网站并不只是一个“展示页面”，它更像是一个可以持续演进的技术名片。

简历更适合快速浏览，而作品集网站更适合把能力表达得更完整。  
尤其是在求职场景里，面试官往往希望快速看到三类信息：

- 你是谁，想做什么方向
- 你做过哪些项目，解决过什么问题
- 你的表达方式是否清晰，工程习惯是否可靠

所以我在做这个网站时，并不是单纯追求页面好看，而是希望它能在很短时间内传达出清晰的职业定位和项目能力。

# 我对网站结构的设计思路

这个网站基于 Next.js App Router 构建，并采用静态导出方式部署。  
选择静态导出的原因很直接：结构简单、部署成本低、适合个人项目，而且在低配置 VPS 上也能稳定运行。

目前网站主要分为几个模块：

- 首页：展示个人介绍、求职方向和技能摘要
- Projects：集中展示项目经历和技术亮点
- Blog：沉淀技术文章和项目思考
- Contact：提供联系方式和进一步沟通入口

这样的结构对个人网站来说已经足够清晰，也方便后续继续扩展。

# 技术选型背后的考虑

我选择 Next.js 的原因，不只是因为它流行，而是因为它在工程组织上非常适合做内容型网站。

一方面，App Router 让页面结构更清晰；另一方面，静态导出可以很好地匹配我的部署需求。  
样式层面我使用 Tailwind CSS，因为它在快速迭代时效率很高，也便于统一页面视觉语言。

在实现过程中，我特别关注以下几点：

- 页面结构是否足够清晰
- 内容是否方便长期维护
- 组件是否可以复用
- 部署流程是否尽量简单

这些问题看起来很基础，但往往决定了项目后续是不是“能持续维护”。

# 我在这个项目里最看重的能力表达

如果只是把简历内容原样搬上网页，这个网站的价值其实有限。  
我更希望它能体现出下面几种能力：

## 1. 信息组织能力

技术能力不只是会不会写代码，也体现在如何组织内容和表达结构。  
一个好的作品集网站，应该让访问者几乎不用思考，就能找到最重要的信息。

## 2. 工程落地能力

我会尽量让页面、数据和样式分层明确。  
比如项目内容使用数据文件驱动，博客内容使用 Markdown 管理，这样后续更新不会影响整体结构。

## 3. 长期迭代意识

我不希望这个网站只是一次性作业。  
相反，我希望它是一个可以不断加入文章、项目和思考的长期项目。

# 下一步我想继续做什么

目前这个网站已经完成了基本展示能力，但我还希望继续完善两类内容：

- 更完整的博客模块，包括文章详情、代码高亮和内容归档
- 更贴近求职场景的项目表达，比如补充项目背景、职责分工和关键结果

我认为，一个好的个人网站不是“堆功能”，而是持续把内容和表达方式打磨得更清楚。  
这也是我做这个网站过程中最有收获的一点。
